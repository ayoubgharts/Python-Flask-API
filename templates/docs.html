API Documentation

Welcome to the API documentation. This guide provides all the necessary information on how to interact with the API.

Base URL

All requests should be made to the following base URL:

http://127.0.0.1:5000

Endpoints

  • GET /api/items

    Retrieve a list of all available items.

  • GET /api/items/<id>

    Retrieve the details of a specific item by its ID. Replace <id> with the actual ID (e.g., /api/items/age).

  • POST /api/items

    Add a new item to the list. The request body should be in JSON format and include an id field (string) and other key-value pairs.

    Example JSON request body:
    {
      "id": "new_id",
      "Name": "New Item Name"
    }

Response Codes

  • 200 OK - The request was successful.
  • 201 Created - A new item was successfully created.
  • 400 Bad Request - The request was invalid (e.g., missing or incorrect data).
  • 404 Not Found - The requested item was not found.

Error Handling

If an invalid URL is accessed, the user will be redirected to a custom error page with an option to go back to the homepage.
